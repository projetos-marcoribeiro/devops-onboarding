## Como preencher a justificativa

A justificativa é o campo mais importante da solicitação. Uma justificativa clara e objetiva acelera a aprovação e evita pedidos de esclarecimento.

Inclua sempre:
- Seu nome e time
- Por que você precisa do acesso
- O que você vai fazer com ele
- Referência a um colega cujo acesso pode ser espelhado (quando aplicável)

**Modelo base de justificativa:**

```
Sou novo no time de DevOps e preciso deste acesso para iniciar minhas atividades.
Em caso de dúvidas, pode espelhar o usuário do meu fellow: [nome do colega].
```

Adapte conforme o sistema e o contexto.

## Exemplos de solicitações

### AWS — Amazon Q Developer

**Sistema:** AWS
**Perfil:** Amazon Q Developer
**Justificativa:**

```
Sou novo no time de DevOps e preciso do acesso ao Amazon Q Developer para
utilizar o assistente de IA durante o desenvolvimento e operação de
infraestrutura na AWS. Em caso de dúvidas, pode espelhar o usuário do
meu fellow.
```

---

### JIRA — Service Desk Users

**Sistema:** JIRA
**Perfil:** Service Desk Users
**Justificativa:**

```
Sou novo no time de DevOps e preciso de acesso ao JIRA para acompanhar
e executar tickets operacionais do time, participar de sprints e
acompanhar o backlog de infraestrutura. Em caso de dúvidas, pode
espelhar o usuário do meu fellow.
```

---

### AWS Teachable — DevOps ReadOnly

**Sistema:** AWS Teachable
**Perfil:** DevOps ReadOnly
**Justificativa:**

```
Sou novo no time de DevOps e preciso de acesso de leitura à conta AWS
do produto Teachable para entender a infraestrutura, acompanhar recursos
e dar suporte operacional ao time de produto. Acesso somente leitura é
suficiente para as minhas atividades atuais. Em caso de dúvidas, pode
espelhar o usuário do meu fellow.
```

## Dicas para não errar

- **Não solicite mais permissão do que precisa.** Comece com ReadOnly quando possível. Você pode solicitar elevação de permissão depois, quando necessário.
- **Solicite tudo de uma vez.** Abra todas as solicitações no primeiro dia para não ficar bloqueado durante o onboarding.
- **Mencione o fellow.** Referenciar um colega cujo acesso pode ser espelhado facilita muito a aprovação.
- **Acompanhe os tickets.** Verifique o status das solicitações no Service Desk. Se passar de 2 dias úteis sem resposta, adicione um comentário pedindo atualização.
